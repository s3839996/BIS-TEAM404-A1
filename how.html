Canvas

Canvas will be the place where assignment updates and specifications are uploaded and released as the project progresses. A private group has also been created for the group which they have been assigned to and this can be used for group discussions and communication.


Trello

Trello is a collaboration tool that will be used to keep track of the project as it progresses. Trello acts like a virtual whiteboard with sticky notes attached that outline tasks that need to be completed as well as their intended completion date. When finished, the task can be marked off.


GitHub

GitHub is another collaborative tool that provides hosting for software development. It will be used to share HTML files for the group website and Java files for programming the project. The repository is setup by one user and the other group members are responsible for committing to the branches and providing evidence of their work on GitHub.

Trello

https://trello.com/b/R5f7Sktt

Trello will be the workspace in which all tasks can be identified, listed, and formally organised. GitHub

https://github.com/s3839996/BIS-TEAM404-A1.git

GitHub will be particularly useful when developing any code-related work, such as the website, or the application itself. GitHub will allow for members to collaboratively work on aspects of these types of work, such as each member developing a separate section of the website or application. A member can then “commit” their work to GitHub, which can then be accessed by other members, as well as serving as a backup in the event of a disaster. All members have used GitHub previously, and are aware how to make commits to the repository.

Word online will be used to collaborate on the final report for the assignment, as well as being used to share and contribute to any other ideas or material. All members are aware how to access Word, as the links have been provided through Microsoft Teams for their convenience.

Group Contract

Expectations

  • Everyone attends the group meetings and actively contributes to discussions.
  • Delegated Tasks should be completed within the allocated timeframe. If assistance is required, each member is expected to raise concerns and support each other.
  • Respect others- listening to everyone's ideas and respecting group members.
  • Members should respond to all messages (especially those where they are mentioned) within the same business day.
  • Members should be open to compromise and be willing to co-operate with other's ideas.
  • Using resources properly- keeping on top of Teams, Trello Boards and participating to all ideas.
  • Meeting and consolidating ideas twice a week.
  • In the event of a member not meeting set expectations, a group meeting should be held to address these issues, and if the problem persists, it should be reported to the tutor, lecturer, as well as being reflected on SparkPlus.
  • Peer feedback throughout the project can be addressed during meetings with the tutor or lecturer, where individuals can raise any concerns.
  • Use Teams for hosting weekly meetings, Trello board should be updated to reflect a members workflow, frequent commits should be made to the GitHub repository.
