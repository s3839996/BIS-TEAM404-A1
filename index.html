    • Project Description

    Finding parking within CBD’s in Australia can be challenging, especially during peak hours. The development of ‘Quick Park’ application will assist users to conveniently park their vehicles amidst all the hustle bustle within the city. Users are required to install the application, register their vehicle, enter their card details, location and get started. The app will highlight all the available parking and indicate the distance from the user to the car park. Users can check the most convenient parking and can select and book the desired parking lot. Once booked, the parking will be reserved for a given duration and the user will have a guaranteed car parking lot available for the selected time period. The user will receive turn by turn instruction on how to get to the carpark. They are also given the option to pre-book a day ahead meaning that the car parking service is allocated for a given time period, but the user must enter and exit the parking lot within strict timings to avoid penalties. Using the app, users can utilise cash less payment system and manage parking fees and extend/shorten carpark time, thereby only paying for the exact amount of time they used the service.

    User Stories

  • “As a manager, I want to save my time so that I don’t arrive late to work”.
  • “As a novice driver, I want to have a secured route with parking so I can avoid getting stressed”.
  • “As a mum of three, I want to find a convenient parking location, so I can drop my children off to kindergarten safely and effortlessly”.
  • Having reserved parking means that you don’t have to worry about being late to work or an important occasion trying to find parking. Just within a few clicks on your mobile device you can enjoy simple and effortless parking options, whilst receiving updates on the vehicle condition and location. The live navigation system will ensure that you’ll never have to go looking for where you parked your car every again. Quick Park application aims to reduce stress for car drivers, finding an affordable and convenient parking within busy business districts in Australia.

    The above tables outline the tasks that are needed to complete Part A of Assignment 1 as well as the workload spread, who’s completing which task and the date that it is due and the group members and their strengths and weaknesses. Each group member previously completed the Introduction to Information Technology course and is aware of the requirements of this project and the commitment required in order for the project to be successful. The table is similar to a Trello Board but has been condensed for presentation purposes. Due to the amount of time it took to decide on a group project, there wasn’t a lot of time to complete the tasks for Assignment 1 Part A. As per the group contract, each group member knows that delegated tasks should be completed within the allocated timeframe. If assistance is required, each member is expected to raise concerns and support each other. The group contract also expresses that resources, such as Canvas, Microsoft Teams, Trello and GitHub, need to be used properly and meetings or status updates must occur at least twice a week. Group members are also aware in the event that they are not meeting set expectations, a group meeting should be held to address these issues, and if the problem persists, it will be reported to the tutor, lecturer, as well as being reflected on SparkPlus. Part of this requires open communication and honesty within a group Before dividing the tasks, the strengths and weaknesses of each group members was declared and from there the tasks were divided best on the strengths and weaknesses so that each assigned task could be played into those strengths. Some of the tasks were assigned to be completed as a group such as the designing of the website, managing the Trello Board and contributing to the group contract whereas other tasks were divided up equally amongst the group members. By doing this, the workload was fairly balanced and there wasn’t too much pressure placed on a single group member. Given that most group members expressed a strength in Java, this was chosen as the primary programming language for our project. The same can be said for HTML, as we all expressed strength with HTML we will all be contributing to authoring and designing the website on GitHub, which will then be moved to AWS. The individual tasks were estimated by the group to be completed in similar timeframes and with a similar time dedication to them, thus each group member was assigned minimum two tasks each. It was also determined that one user would be responsible for setting up the Git Repository and AWS Site, whilst all users are comfortable with using these services, it was more convenient to have one group member set it up and the others would contribute as to avoid errors with permissions.
